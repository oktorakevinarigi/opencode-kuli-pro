## Usage

### Invoking Agents

You can invoke these agents in two ways:

1. **Manual invocation** - Use `@mention` syntax:
   ```
   @frontend-developer help me build a Next.js 14 app with shadcn/ui
   @security-auditor review this authentication code
   @python-expert optimize this data processing script
   ```

2. **Automatic invocation** - Primary agents may automatically call subagents based on the task requirements.

### Agent Configuration

Each agent is configured with:

- **Mode**: All agents are set to `subagent` mode
- **Tools**: Appropriate tool access based on the agent's function
- **Permissions**: Security-conscious permission settings
- **Temperature**: Optimized for the agent's specific use case

### Tool Access Patterns

Agent tool access varies based on their function:

- **Full Access** (Read/Write/Edit/Bash): Development, debugging, and implementation agents
- **Read-Only** (Read only): Review, analysis, and security assessment agents  
- **Selective Access**: Based on agent-specific requirements and security considerations

Each agent's tool configuration is optimized for their specific use case while maintaining security best practices.

## Example OpenCode Configuration

You can also configure agents in your `opencode.json`:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "agent": {
    "custom-review": {
      "description": "Custom code review agent",
      "mode": "subagent",
      "tools": {
        "read": true,
        "write": false,
        "edit": false
      }
    }
  }
}
```

## Integration with OpenCode

These agents integrate seamlessly with OpenCode's agent system:

- Use Tab key to switch between primary agents
- Mention subagents with `@agentname`

- Monitor usage and optimize configurations
